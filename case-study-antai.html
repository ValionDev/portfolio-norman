Case Study

ANTAI Web Reader

Designing the user flow and interface for a responsive, accessibility-focused web reader, a content platform built for reading and publishing experiences.

Role

UI Designer

Client

ANTAI

Timeline

July 2023

Scope

User flows, UI design, illustrations

Platform

Responsive web

Overview

ANTAI needed a web-based reading platform focused on accessibility and a clean reading experience. The project was a one-month engagement that required rapid design delivery, from user flow mapping through final UI and custom illustrations.

This project bridges my product design work with the content/publishing domain, designing for readers, authors, and the experience of consuming long-form content on the web.

The Challenge

Content platforms face a fundamental tension: they need to serve both readers (who want distraction-free, comfortable reading) and publishers (who want their content to reach and engage audiences). The interface must get out of the way for reading while still being discoverable and navigable.

Accessibility was a non-negotiable requirement. The platform needed to work for users with diverse needs, screen readers, keyboard navigation, adjustable typography, and high-contrast modes.

Approach

  • User flow mapping: Charted the complete reading and publishing journey, identifying key decision points and potential friction
  • Responsive design: Designed for seamless reading across devices, with particular attention to mobile reading comfort (typography scale, touch targets, scroll behavior)
  • Accessibility-first: Built accessibility into the design from the start, not as an afterthought, semantic structure, contrast ratios, keyboard flows, screen reader compatibility
  • Custom illustrations: Created original illustrations that gave the platform a distinct visual identity

Key Design Decisions

Reading Experience

The reading view prioritizes typography and whitespace. I chose a comfortable line length (65-75 characters), generous line height, and a clean serif typeface for body text. The chrome fades away while reading, navigation and controls are accessible but not competing for attention.

Accessibility Features

The platform includes adjustable font sizing, a high-contrast mode, and full keyboard navigation. All interactive elements have visible focus states, and the information hierarchy works with assistive technology, not just visually.

Outcome

Delivered the complete design within the one-month timeline, user flows, responsive UI, accessibility specifications, and custom illustrations. The project demonstrated that accessibility-first design doesn't mean compromising on visual quality; the constraints actually led to cleaner, more intentional design decisions.
